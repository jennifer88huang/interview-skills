## 一、产品概述

### 1.1 产品背景

大厂面试备考长期存在三个核心痛点：

1. **面经失真**：网络面经是别人的经历，和自身简历、目标 JD 脱节，背了不一定能用上
2. **准备无焦点**：不知道面试官会从哪切入，容易泛泛而备、重点错位
3. **缺乏真实演练**：找不到合适的"陪练"，只能自己想象问答，缺乏追问和压力感

**interview-skills** 是一款 AI Skill，以候选人自己的简历和目标岗位 JD 为输入，生成专属面试题并模拟真实追问过程，解决"备考没有针对性"的核心问题。

### 1.2 产品定位

> **你的专属 AI 大厂面试官**
> 不是题库，是基于你的简历 + 真实 JD 的个性化面试模拟器。

### 1.3 目标用户

| 用户类型 | 描述 | 核心诉求 |
|---------|------|---------|
| 主力用户 | 正在准备大厂面试的求职者（应届 / 社招） | 针对自身情况快速生成专属面试题 |
| 次要用户 | 在职人员，预判跳槽 / 晋升 P 级面试方向 | 了解目标岗位考察点，提前对齐 |
| 延伸用户 | 面试官 / HR，用来校验候选人简历质量 | 快速识别候选人简历亮点与弱点 |

## 三、功能范围

### 3.1 核心功能（v1.0 已上线）

#### F1：JD 智能解析
- 输入：用户粘贴的 JD 原文（支持不完整）
- 输出：结构化需求清单（硬技能 / 软技能 / 隐藏考察点）
- 关键能力：识别"字里行间没写但其实在考"的隐性要求

#### F2：简历智能解析
- 输入：简历文字 / PDF / 图片
- 输出：技术栈、项目亮点、简历弱点（描述过浅 / 易被卡壳）
- 关键能力：数字挖掘、时间轴分析、经历匹配度评估

#### F3：JD vs 简历匹配度分析
- 输出三维评估：
  - ✅ 强匹配（候选人优势区）
  - ⚠️ 需补强（有基础但不充分）
  - ⚠️ 简历弱点（可能被面试官重点追问）

#### F4：公司面试风格匹配
- 支持公司：阿里 / 字节 / 腾讯 / 百度 / 美团 / 京东 / 华为 / 滴滴 / 拼多多 等
- 每家公司对应独立的出题风格、题型比例、文化偏好
- 示例：阿里 → 重业务理解 + 六脉神剑；字节 → 算法必考 + impact 导向

#### F5：专属面试 10 问生成
- 题型构成（技术岗默认比例，按岗位自动调整）：
  - 技术/专业硬实力题 × 3-4
  - 项目深挖题 × 3-4
  - 行为/价值观题 × 1-2
  - 反问预测题 × 1
- 每题附：难度 ⭐ + 考察点 + 参考答案方向 + 追问路径

#### F6：互动模拟追问
- 用户回答任意一道题后，可说「帮我追问」
- AI 扮演面试官，基于用户的作答继续深挖
- 模拟真实面试压力，直到用户满意为止

#### F7：好答案 vs 差答案对比示例
- 用户可针对任意题目说「给我 Q3 的好答案 vs 差答案」
- 输出同一道题的低分示例、高分示例、逐项点评、追问点和临场作答模板
- 按题型区分为两套模板：技术题版 / 行为题版
- 好答案需贴合岗位、公司风格、JD 要求与用户简历；无简历时按理想候选人示例生成
- 差答案需模拟真实候选人的常见失分方式：空泛、无结构、无案例、经不起追问

#### F8：HR 面专项训练模块
- 用户可主动说「模拟HR面」「只练HR面」「帮我练离职原因/薪资谈判/职业规划」
- 输出 6-8 道 HR 专项题，而不是通用技术/业务混合题
- 默认覆盖 4 类高频主题：求职动机、职业稳定性、薪资预期、职业规划
- 每题需包含：HR 真正想判断什么、回答原则、踩坑提醒、追问方向
- 对频繁跳槽、空窗期、转行、薪资涨幅过高等风险点，需追加针对性问题

#### F9：备战建议输出
- 基于匹配度分析，给出 3 条分级建议
- 分级：【紧急】/ 【参考】
- 示例：「【紧急】准备具身基座模型基础概念，面试命中概率 60%+」

### 3.2 面试轮次支持

| 轮次 | 出题风格 | 时长预估 |
|------|---------|---------|
| 一面 | 基础知识 + 算法 + 简历1-2个项目 | 45-60 min |
| 二面 | 系统设计 + 项目深挖 + 技术选型 | 45-60 min |
| 三面 | 综合能力 + 价值观 + 职业规划 | 30-45 min |
| HR 面 | 稳定性 + 薪资预期 + 文化匹配 | 20-30 min |

### 3.3 岗位覆盖

| 类别 | 岗位 |
|------|------|
| 技术类 | 后端 / 前端 / 移动端(iOS/Android) / 算法/AI / 数据工程 / 架构师 / SRE / 安全 |
| 产品类 | 产品经理(C端/B端/增长/游戏) |
| 商业类 | 运营 / 市场营销 / 商业分析 |
| 职能类 | 数据分析师 / HR / 法务 / 财务 |

### 3.4 管理命令（用户指令集）

| 用户输入 | 触发行为 |
|---------|---------|
| `重新出题` / `换一批` | 重新生成 10 道题 |
| `模拟一面/二面/三面` | 按指定轮次风格出题 |
| `模拟HR面` / `只练HR面` | 进入 HR 面专项训练模块 |
| `帮我练离职原因 / 薪资谈判 / 职业规划` | 只练指定 HR 主题 |
| `帮我详细解答Q[N]` | 深入展开某题参考答案 |
| `再出5道题` | 扩充题库 |
| `帮我追问` | 进入追问模拟模式 |
| `给我Q[N]的好答案 vs 差答案` | 查看该题高分/低分对比示例 |
| `列出支持的公司` | 查看公司列表 |

## 四、Skill 技术架构

### 4.1 文件结构

```
interview-skills/
├── SKILL.md                    # 主入口，完整面试流程指令集
├── README.md                   # 用户使用文档
├── PRD.md                      # 本文档
└── references/
    ├── company-profiles.md     # 各大厂面试风格、简历筛选标准、完整流程
    ├── question-design.md      # 题目设计原则、各岗位题库框架
    ├── jd-parser.md            # JD 解析策略（隐藏考察点识别）
    ├── resume-parser.md        # 简历解析策略（数字挖掘 / 时间轴分析）
    └── bei-framework.md        # 行为面试 BEI/STAR 框架 + 各厂价值观题库
```

### 4.2 核心处理流程

```
用户输入（公司 + 岗位 + JD + 简历）
        │
        ├──→ [F1] 解析 JD → 结构化需求清单
        ├──→ [F2] 解析简历 → 技术栈 + 亮点 + 弱点
        │
        ▼
  [F3] 匹配度分析（✅强匹配 / ⚠️需补强 / ⚠️简历弱点）
        │
        ▼
  [F4] 加载公司风格配置（company-profiles.md）
        │
        ▼
  [F5] 生成面试 10 问（question-design.md）
        │
        ▼
  [F9] 输出备战建议
        │
        ├──→ [F6] 可选：互动追问模式（用户触发）
        ├──→ [F7] 可选：好答案 vs 差答案对比（用户触发）
        └──→ [F8] 可选：HR 面专项训练（用户触发）
```

## 五、用户体验设计

### 5.1 核心用户旅程

```
发现 Skill Store 中的 interview-skill
        ↓
一键安装，无需配置
        ↓
对话框输入：「我要面阿里，技术运营官，帮我模拟面试」
        ↓
AI 引导输入 JD + 简历（粘贴/上传均可）
        ↓
获得匹配度分析（30秒内）
        ↓
获得专属面试 10 问（含完整信息）
        ↓
逐题作答 → 触发「帮我追问」→ 深度模拟
        ↓
获得备考建议，有针对性地继续准备
```

### 5.2 输入容错设计

| 场景 | 处理方式 |
|------|---------|
| 只有公司+岗位，无 JD | 基于理想候选人画像出题，注明「基于行业通用 JD 推断」 |
| 只有 JD，无简历 | 基于 JD 理想候选人模型出题，注明「建议补充简历以获得个性化题目」 |
| JD / 简历不完整 | 提取可用信息继续执行，不卡流程 |
| 公司不在支持列表 | 根据 JD 内容推断公司风格，提示「该公司暂无专项配置，已根据 JD 自动推断」 |

### 5.3 输出规范

每次完整输出包含以下模块（顺序固定）：

```
1. 面试报告头（公司 · 岗位 · 面试官身份）
2. 公司风格分析（4条，口语化）
3. JD vs 简历匹配度分析（三维）
4. 面试 10 问（统一格式，含难度+考察点+答题方向+追问）
5. 备考建议（3条，分级）
6. 继续备战指引（互动提示）
7. 可选：某题「好答案 vs 差答案」对比示例
8. 可选：HR 面专项训练题组
```

## 七、已知限制与风险

| 限制 / 风险 | 说明 | 缓解措施 |
|------------|------|---------|
| AI 幻觉风险 | 参考答案方向可能出现不准确的技术描述 | 明确标注「参考方向」而非标准答案，鼓励用户结合实际验证 |
| 公司风格漂移 | 各大厂面试风格可能随时间变化 | references/company-profiles.md 持续迭代更新 |
| 简历解析精度 | PDF 中图片/表格内容解析可能丢失 | 建议用户补充文字说明 |
| 岗位覆盖不全 | 部分细分岗位（如芯片、嵌入式）题库较薄 | 路线图中规划专项扩充 |
| 追问深度限制 | 单次上下文有限，长时间追问可能丢失前文细节 | 建议用户分段模拟，每次聚焦 2-3 道题 |
| 示范答案误导风险 | 用户可能直接背诵示例，导致表达僵硬 | 输出保持口语化，并附“结构模板”而非统一话术 |
| HR 回答过度包装风险 | 用户可能把回答修饰得过于标准化，显得不真诚 | 强调“可说版本”而非“标准背稿”，保留真实动机和边界 |

## 十一、F7 详细设计

### 11.1 触发方式

用户在已有题目上下文中可直接输入：

```text
给我 Q3 的好答案 vs 差答案
Q5 怎么答才算好答案？
给我这个问题的差答案示例
```

若用户未指定题号，但上下文明确指向最近一题，则默认基于最近讨论题目生成。

### 11.2 输出结构

#### A. 技术题版

```text
1. 题目回顾
2. 差答案示例
3. 为什么这是差答案（3点）
4. 好答案示例
5. 为什么这是好答案（3点）
6. 面试官可能继续追问
7. 临场作答模板
```

#### B. 行为题版

```text
1. 题目回顾
2. 差答案示例
3. 为什么这是差答案（3点）
4. 好答案示例
5. 为什么这是好答案（3点）
6. 面试官可能继续追问
7. 临场作答模板（STAR/BEI）
```

### 11.3 生成原则

- 技术题必须使用技术题版模板：强调结论先行、排查或设计框架、关键权衡、真实项目证据
- 行为题必须使用行为题版模板：强调 STAR/BEI 结构、个人贡献、冲突与结果复盘
- 差答案不能写成明显错误答案，而要模拟“懂一点但答不深”的真实失分状态
- 好答案避免模板腔，保留口语化表达，降低用户死记硬背风险
- 混合题（如“技术推动 + 跨团队协作”）默认按主考察维度选模板，另一维度作为好答案中的补充亮点展开

### 11.4 验收标准

- 用户指定任意题号后，能生成结构完整的正反示例
- 好答案和差答案必须围绕同一题目，不得答非所问
- 点评必须指出至少 3 个具体优缺点，不能只给泛泛评价
- 输出中必须包含至少 2 个可继续追问的问题
- 技术题输出必须含“结论/原理或排查框架/项目或场景证据/权衡或结果”中的关键元素
- 行为题输出必须含“背景/目标/动作/结果/复盘”中的关键元素

## 十二、F8 详细设计

### 12.1 触发方式

用户可直接输入：

```text
模拟HR面
只练HR面
帮我练离职原因
帮我练薪资谈判
帮我练职业规划
```

若用户明确说“下轮是 HR 面”或“先不练技术题”，也默认进入该模块。

### 12.2 输出结构

```text
1. HR 关注重点（3点）
2. HR 专项题组（6-8 题）
3. 每题的隐藏考察点
4. 每题的回答原则
5. 每题的踩坑提醒
6. 每题的追问方向
7. HR 面回答建议（3条）
```

### 12.3 生成原则

- 社招优先覆盖：离职原因、职业稳定性、薪资预期、岗位匹配、入职时间
- 校招优先覆盖：求职动机、offer 选择、地点接受度、长期发展、稳定性
- 对用户简历中的风险点要主动加题，例如频繁跳槽、空窗期、转行、涨薪预期过高
- 回答建议强调“真诚但有边界”，不是教用户虚构经历
- 对薪资题要给区间策略和谈判边界，不要只给单点数字

### 12.4 验收标准

- 用户进入 HR 面模式后，不再输出通用技术 10 问
- 题组至少覆盖 3 个 HR 高频主题
- 每题都必须包含“HR 真正想判断什么”与“踩坑提醒”
- 对明显高风险背景，输出中必须出现至少 1 道定向问题
