# 行为面试（BEI / STAR）框架说明

> 用于设计 Q8-Q9 行为类题目，以及在互动追问模式中评估用户回答质量。

---

## 什么是 BEI / STAR

**BEI（Behavior Event Interview）行为事件面试**：通过追问候选人过去真实行为来预测未来表现。

**STAR 法则**（回答框架）：
- **S**ituation — 背景：当时的情境是什么？
- **T**ask — 任务：你的职责/目标是什么？
- **A**ction — 行动：你具体做了什么？
- **R**esult — 结果：最终结果/数据/影响是什么？

## 各大厂价值观对应题库

### 阿里巴巴（6大价值观）

| 价值观 | 对应面试题 |
|--------|-----------|
| 客户第一 | 「描述一次你为了用户体验，推翻了技术/业务同学意见的经历」 |
| 团队合作 | 「你经历过最难推动的跨团队项目是什么？你是怎么做的？」 |
| 拥抱变化 | 「你在什么情况下放弃了自己坚持的方案？你怎么适应的？」 |
| 诚信 | 「你有没有在压力下妥协做过一个不认同的决策？结果如何？」 |
| 激情 | 「你在业余时间会主动做什么和工作相关的事？」 |
| 敬业 | 「说一个你为了项目牺牲了什么的经历」（注意 Work-life balance 边界） |

### 字节跳动（6大价值观）

| 价值观 | 对应面试题 |
|--------|-----------|
| 志存高远 | 「你对 [行业] 5年后的格局有什么判断？你打算怎么准备？」 |
| 务实敢为 | 「说一个你做了一个别人不看好但最终证明正确的决策」 |
| 开放谦逊 | 「你收到过让你印象最深的一次负面反馈，你怎么回应的？」 |
| 坦诚清晰 | 「你有没有因为说了实话让别人不开心的经历？怎么处理的？」 |
| 始终创业 | 「你在现有工作中有没有主动推动过一个不在职责范围内的改进？」 |
| 多元兼容 | 「你和你最不同意见的同事是怎么合作的？」 |

### 腾讯
- 「说一个你发现系统隐患并主动推动修复的案例」
- 「你遇到过需求不合理的情况怎么处理？」
- 「你最近一次的晋升/绩效复盘，你觉得自己哪里还不够好？」

### 美团
- 「描述一个你用数据推动业务决策的案例」
- 「你如何处理执行过程中出现的资源紧缺问题？」

## BEI 题追问逻辑

当用户在互动模式回答了 BEI 题时，按以下逻辑追问：

1. **S/T 不清楚** → 「当时背景能说清楚一些吗？这是在什么公司/团队/阶段？」
2. **A 不具体** → 「你具体做了什么？是你主导还是和其他人一起？」
3. **R 没数据** → 「最终结果如何？有没有可以量化的指标？」
4. **全用「我们」** → 「你在这件事里个人贡献了什么？」
5. **回答过于完美** → 「这件事有没有什么遗憾或者你觉得没做好的地方？」

## 评分维度（互动追问模式使用）

在模拟面试互动模式中，对用户回答的评估维度：

| 维度 | 优秀表现 | 不足表现 |
|------|---------|---------|
| 结构性 | STAR 完整，逻辑清晰 | 散乱，跳跃 |
| 具体性 | 有数字、名字、时间 | 全是模糊表达 |
| 影响力 | 结果有业务影响 | 结果不清晰 |
| 个人贡献 | 「我」字明确 | 「我们」模糊化 |
| 反思深度 | 有总结和教训 | 只说结果 |
