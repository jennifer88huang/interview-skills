# 面试题设计原则与题库框架

> 本文件指导 SKILL.md Step 3 如何设计高质量面试题。

---

## 核心设计原则

### 1. JD 驱动
每道题都要能对应 JD 中的至少一个要求或加分项。在题目后标注「📌 考察点」时，引用 JD 原文关键词。

### 2. 简历锚点
至少 3 道题要与简历中的具体项目/经历挂钩，让候选人感受到「面试官真的看过我简历」。
格式：「你简历上提到在 XX 公司做了 XX 项目，当时……」

### 3. 由浅入深
题目排序遵循：破冰题（Q1）→ 专业基础（Q2-Q4）→ 项目深挖（Q5-Q7）→ 行为/价值观（Q8-Q9）→ 收尾/反问（Q10）

### 4. 追问预设
每道题配备 1~2 个追问方向，便于用户模拟真实面试的「被追问」感。

## 各岗位题型框架

### 技术类（后端 / 前端 / 移动端 / SRE）

| # | 类型 | 示例 |
|---|------|------|
| Q1 | 破冰/自我介绍切入 | 「请结合你最近的一个项目介绍一下自己」 |
| Q2 | 基础原理 | 数据结构、网络、OS、数据库原理 |
| Q3 | 语言/框架深度 | Java GC 机制 / Vue 响应式原理 / Swift ARC |
| Q4 | 算法（字节必考）| LeetCode Medium 场景题 |
| Q5 | 系统设计 | 「如何设计 XX 系统，支持 XX 规模」 |
| Q6 | 项目深挖 | 「你简历上的 XX 项目，最大技术挑战是什么」 |
| Q7 | 故障/复盘 | 「你遇到过最难排查的线上问题是什么」 |
| Q8 | 协作/软技能 | STAR 法则，跨团队推动 |
| Q9 | 价值观/文化 | 公司价值观对应题，见 BEI 框架 |
| Q10 | 职业规划 | 「3年后你希望在什么位置，为什么选我们」|

### 算法/AI 岗

| # | 类型 | 示例 |
|---|------|------|
| Q1 | 研究方向介绍 | 「介绍你做过的最有挑战的一个 AI 项目/论文」 |
| Q2 | 数学基础 | 概率统计、线性代数、凸优化基础题 |
| Q3 | 模型原理 | Transformer / BERT / LLM 推理优化等 |
| Q4 | 工程落地 | 「如何把模型部署到线上，如何做效果评估」 |
| Q5 | 特征工程 | 推荐/广告场景特征构建 |
| Q6 | 论文追问 | 「你说你复现了 XX 论文，遇到了什么问题」 |
| Q7 | 数据处理 | 「如何处理样本不均衡问题」 |
| Q8 | 业务场景设计 | 「设计一个短视频推荐系统」 |
| Q9 | 行为/合作 | STAR 题 |
| Q10 | 职业规划 | 学术路还是工业路 |

### 运营岗

| # | 类型 | 示例 |
|---|------|------|

| Q7 | 商业化 | 「你做过最有创意的商业合作或变现尝试」 |

## 题目难度分级（按目标职级）

| 职级 | 难度定位 | 说明 |
|------|---------|------|
| 应届/P4 | 基础扎实 | 数据结构/算法/框架基础，项目说清楚 |
| P5 | 有深度 | 能独立解决复杂问题，有一定系统设计能力 |
| P6 | 引领级 | 架构能力、跨团队影响力、业务 sense |
| P7+ / 管理岗 | 战略级 | 业务战略判断、团队建设、上下对齐 |

## 输出质量 checklist

生成 10 问前确认：
- [ ] 至少 3 题锚定简历具体经历
- [ ] 至少 2 题直接对应 JD 关键要求
- [ ] 包含至少 1 道行为面试（BEI/STAR）题
- [ ] 包含至少 1 道开放性思维题
- [ ] 题目语气符合目标公司面试风格
- [ ] 每题配备追问方向
